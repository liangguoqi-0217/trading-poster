SAP QM 检验结果复制与 UD 复制机制

标准功能 Copy Inspection Results 可以复制检验结果，但 UD（使用决策）不能简单复制 —— 两者在 SAP 设计上是分开的
核心原则：检验结果可以参考，但质量决策必须由质量人员针对当前批次确认

二复制结果的前提条件

1MIC 一致

目标检验批必须已存在对应 MIC（Master Inspection Characteristic），SAP 不会自动新增检验项目。

源：含量、水分、杂质
目标：含量、水分
→ 杂质不会被复制（目标没有该 MIC）

2Origin Result Data 配置

需在检验计划的 MIC 中维护 Origin Result Data 字段：

01 Manual entry（手动录入）
02 Previous inspection lot（历史检验批）

如需复制历史结果，一般配置为 02

3复制的是结果，不是检验批

复制后目标检验批仅获得检验结果数值，以下信息不会复制：

检验日期 检验员 样品 缺陷 UD
SAP 设计理念：每一个检验批都是独立的质量记录

三复制后：检验批状态对比

源检验批 10000001

REL RREC UD ✓
已完成 · 库存已转非限制
→
复制结果

目标检验批 10000002

REL RREC 无 UD
结果已复制 · 但仍需人工做 UD
结果复制过来了，但 UD 不会自动跟随 —— 这是 SAP 的刻意设计

四UD 能不能复制？

标准 SAP：不能直接复制 UD

SAP 没有类似 Copy Usage Decision 的标准功能。

1UD 触发库存移动

  • 质量库存 → 321 → 非限制库存
  • 质量库存 → 350 → 冻结库存
  • 不合格 → 122 → 退货供应商
关键：库存移动不可自动触发
每次库存移动必须有据可查的决策人

2UD 触发后续动作

  • 自动过账（财务凭证）
  • 质量评分（Vendor Evaluation）
  • 批次状态变化（Batch Status）
  • 后续通知单（Follow-up Actions）
关键：这些动作必须有明确责任人
直接复制 UD 会导致审计追溯链断裂
SAP 设计理念：检验结果可以参考，但质量决策必须由质量人员针对当前批次确认

五实际业务中怎么实现"自动UD"？

方案1 · 推荐 自动 UD

SAP 标准功能，根据规则自动执行 UD。

条件：所有 MIC 合格 + 无缺陷
↓
系统自动 UD（UD 代码：A1 合格）
↓
321 移动（质量 → 非限制库存）
适合：重复检验 / 稳定性测试 / 高频物料

方案2 复制结果 + 自动 UD

药企常用模式，将两个标准功能组合使用。

旧检验批
↓ 复制结果
新检验批
↓ 自动 UD
库存释放

典型场景：稳定性考察（0M / 3M / 6M / 12M），各时间点复制结果 + 自动 UD，减少人工重复操作。

方案3 开发增强（不推荐）

技术上可读取 QALS / QAMR / QAVE 复制 UD 代码，但存在严重风险：

审计风险：FDA/NMPA 关注"谁做的 UD？基于什么数据？"
合规风险：两个批次质量决策完全一致但没有人工确认
推荐策略：优先使用方案 1（自动 UD 规则）+ 方案 2（复制结果组合），避免方案 3（直接复制 UD 增强），确保审计追溯链完整。

六结合药企 MES 场景的建议

MES 层实现"检验结果继承"（而非复制 UD）

历史检验批
已完成 UD
→
结果模板
提取 MIC 数据
→
新检验批
自动带入建议值
→
QA 确认
人工审核+修订
→
执行 UD
责任明确
QA 人工确认环节不可省略 —— 数据可以继承，但质量责任不能自动转移
药企质量体系核心：检验结果可以参考历史，但 UD 质量决策必须由当前责任人针对当前批次确认
给 MES 设计的启示：如果你后续准备启用 SAP QM 或让 MES 模仿 SAP QM，建议重点研究"检验批 → 检验特性 → 结果记录 → UD"四者之间的数据模型，这比单纯研究事务码更重要。
总结：SAP QM 中检验结果可以复制（Copy Inspection Results），但 UD（使用决策）不能直接复制 —— 因为 UD 不是单个字段，而是触发库存移动、财务过账、质量评分、批次状态变化的业务决策链条。 药企场景下，推荐在 MES 层实现"检验结果继承 → QA 确认 → UD"的流程，确保数据可继承，责任不转移。
