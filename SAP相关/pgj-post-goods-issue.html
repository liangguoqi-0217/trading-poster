三PGI vs 容易混淆的概念

PGI ≠ 出库 ≠ 开票 ≠ GI

动作业务含义库存变化财务影响典型事务
交货单创建通知仓库准备发货❌ 无❌ 无VL01N
拣配从货架取出货物⚠️ 移库❌ 无LT03
PGI（发货过账）货物正式离开企业库存，责任转移✅ 减少✅ 成本结转VL02N
开票 Billing向客户收费❌ 无✅ 收入确认VF01
GI（发料）生产领料（非销售）✅ 减少✅ 生产成本MB1A / 移动 261
PGI 属于销售物流（移动类型 601）；GI（261）属于生产物流。两者都减少库存，但业务场景和会计科目完全不同。

四没有 PGI 规范，各部门痛在哪？

销售部门 SD

订单承诺与实际发货脱节：不知道货是否发出、发了多少
PGI 后：订单状态透明（已交货 / 已 PGI / 未交货），避免超卖

仓库部门 物流

实货已装车离厂，系统库存还在 → 账实不一致
PGI 实现：实物动作 → 系统动作 → 财务动作 三者一致

财务部门 FI

收入与成本时间不匹配：货已发出但库存还在账上，资产虚高
PGI 自动触发：借主营业务成本 / 贷库存商品，账务同步

成本会计 CO

不知道销售成本何时发生，毛利分析不准
PGI 时成本从生产库存转入销售成本，CO 可分析毛利

质量部门 QM

未放行库存可能被销售发货 → GMP 合规风险；召回时无法定位批次
QM 状态控制：未放行不允许 PGI；PGI 记录完整批次追溯

生产部门 PP

销售端库存不准 → MRP 运算失真 → 生产计划错误
PGI 后库存准确 → MRP 可靠 → 计划更准

管理层视角

库存准确率

PGI 提高账实一致，减少月底大盘调整

应收及时性

PGI → 开票 → 应收，链条缩短，加速回款

客户服务水平

从订单量 → 交付量 → 准时交付率，全链可追踪

五为什么 SAP 设计 PGI 这个独立步骤？

① 订单 ≠ 实际发货

客户订 1000 箱，库存可能不足，不能直接减少库存。需要交货环节确认实际可发数量。

② 发货前有物流过程

客户订 1000 箱，仓库实际只能发 800 箱。PGI 只确认实际发生的数量。

③ 需要库存责任切换

货物从"企业库存"变成"在途/客户资产"，必须有明确的系统时间点记录责任转移。

六PGI 常见问题与解决

Q1: 交货单做了，为什么库存没减少？
因为只创建了交货单，还没执行 PGI。检查 Goods Movement Status 是否仍为 "A"（未过账）。必须通过 VL02N 执行 PGI 才会减少库存。
Q2: 为什么 PGI 报错做不了？
常见原因 4 种：
① 批次库存不足 ② 库存地点选错 ③ 过账期间已关闭 ④ QM 检验库存未放行（药企常见：Q 库存不能直接销售）
Q3: PGI 错了能取消吗？
可以。事务码 VL09（取消发货），产生反向移动类型 602，库存回来。不要直接改表，保证审计完整性。
流程：VL09 → 修改交货 → 重新 PGI
Q4: 药企 QM 库存如何限制 PGI？
启用 QM 后：质量检验库存（Q 库存）→ UD 放行 → 非限制库存 → 才允许 PGI。没有放行则系统自动阻止发货，避免不合格品流向客户。

七PGI 最佳实践：不是"所有交货必须 PGI"就完了

① 明确 PGI 责任人

仓库负责 PGI，不要由销售执行。原则：谁控制实物流，谁确认发货。

② 控制 PGI 前置条件

必须检查：库存是否充足 → 批次是否指定 → 质量是否放行 → 数量是否超过订单。药企尤其必须锁住质量状态。

③ 规范异常处理

错误发货 ≠ 直接改表。标准路径：VL09 取消 PGI → 修改交货单 → 重新 PGI，保证审计追溯完整。

核心理由：库存价值、财务凭证、批次追溯最终应由 SAP 控制。MES 可以驱动发货动作，但"企业库存正式减少"这一事件必须由 ERP 的 PGI 来确认。
从产品经理角度看：PGI 和你设计的 MES 投料、报工、收货逆向操作理念本质相同——每一个关键业务事件，必须有明确的状态转换 + 财务/库存后果。这是 SAP 产品架构的统一哲学。

九一句话总结

PGI 解决的不是"仓库发货操作"，而是解决企业从客户订单到库存、财务、质量追溯之间的信息断点。它是连接 SD / MM / FI / CO / QM 的关键业务事件节点——当 PGI 发生时，SAP 正式承认这批货已离开企业，库存责任、成本责任、追溯责任同步转移。
