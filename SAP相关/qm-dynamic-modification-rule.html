SAP QM 动态修改规则 DMR：质量策略自动化设计

Dynamic Modification Rule — 根据历史质量表现动态调整检验强度，实现「质量策略自动化」
核心理念：质量控制不是固定流程，而是根据历史质量表现动态调整检验强度

一先理解业务模型

典型场景 — 采购检验动态调整

以物料 A疫苗原液 + 供应商 100001 为例：

每批全检（新供应商首次供货）

系统自动判断：当前供应商 + 当前物料 → 历史质量表现 → 下一次采购是否需要检验？检验等级是多少？

四质量等级的唯一性设计 — 关键理解点

质量等级不是简单的「物料」单一维度，而是基于多维度组合的唯一键：

组合维度说明示例
物料 + 供应商同一物料不同供应商独立计分A疫苗 / 供应商A：100批合格
A疫苗 / 供应商B：5批不合格
物料 + 供应商 + 客户按客户区分质量要求不同客户可能有不同质量标准
物料 + 制造商自制 vs 外购区分自制批次 vs 外购批次
物料 + 批次按批次粒度跟踪特定批次的质量历史
同物料，供应商 A（100 批合格）和供应商 B（5 批不合格）的 DMR 质量等级不会混在一起

七运行逻辑 — DMR 完整生命周期

场景：供应商第一次送货，物料 A疫苗原液

Step 1：收货 MIGO 101
生成检验批 10000001
↓
Step 2：系统检查质量等级
读取 QAVE（物料 + 供应商）→ 无历史记录 → 默认阶段 10（加强检验）
↓
Step 3：创建检验规格
根据 DMR 阶段 10 → 决定：全检 + 标准抽样
↓
Step 4：QC 检验 + UD
QA11 执行 UD：合格 → 系统更新 QAVE：当前阶段 + 合格次数 +1
↓
Step 5：下一批
系统重新读取 QAVE → 判断：是否符合升级条件？
升级逻辑：连续合格 N 批 → 自动提升阶段 → 降低检验强度
降级逻辑：出现不合格 → 立即降回加强阶段 → 全检模式

MES 调用逻辑

1. 创建检验批
↓
2. 查询质量等级（Z_QM_LEVEL: 物料 + 供应商）
↓
3. 匹配动态规则（Z_QM_DMR: 当前阶段 → 抽样方案）
↓
4. 决定检验方案（全检 / 抽检 / 免检）
↓
5. 产生检验任务（分配给 QC 人员）
↓
6. UD + 更新质量等级（Z_QM_LEVEL: 合格 +1 / 不合格 +1）
未来扩展：结合 MES 已有的设备、人员、物料、批次全生命周期数据，可进一步升级为「质量风险等级引擎」—— 本质就是 SAP DMR 的升级版，纳入更多维度（设备状态、环境参数、人员资质等）作为规则输入。

十产品经理 TAKEAWAYS — SAP DMR 最值得学习的设计思想

01

配置驱动，非代码驱动

业务规则变更（如 10 批→20 批）只需修改配置参数，无需开发。DMR 的「阶段 + 检验点」本质是一个可配置的状态机。

02

业务对象严格分离

检验事务（Lot）、检验结果（Result）、质量历史（Level）、规则引擎（DMR Rule）各司其职，互不耦合。每层可独立扩展。

03

历史数据驱动未来流程

过去质量表现 → 未来质量策略，这是非常先进的设计。同样适用于供应商评分、采购策略、库存策略、风险控制等场景。

这套「配置驱动的状态机 + 历史数据驱动的策略切换」思想，不仅适用于 QM，也适用于供应商评分、采购策略、库存策略、风险控制等多个领域
总结：SAP QM 动态修改规则（DMR）的核心是「质量等级（QAVE）+ 规则引擎（QDQL）」的组合设计。 它通过检验批（QALS）触发、质量等级（QAVE）查询、DMR 规则（QDQL）匹配、结果（QAMR）记录、 UD（QAVE/QAMB）闭环的完整链路，实现了质量策略的自动化和动态化。 MES 设计中应借鉴其「配置驱动 + 对象分离 + 历史驱动未来」的架构思想， 构建独立的质量等级表和动态规则引擎，而非把所有逻辑耦合在一张表里。
