01 · MARD 与 MARDH 的关系
MARD · 库存地点库存（当前）
属性说明
数据时间当前期间
更新频率每次收发货实时更新
数据特点只保留最新一条库存状态
典型用途查当前库存、MRP运算
关键事务MB52 / MMBE / MMB1
MARDH · 库存地点库存历史（月末快照）
属性说明
数据时间历史期间
更新频率跨期间库存变化时写入一次
数据特点保存期间切换时的库存快照
典型用途月末库存分析、库存趋势
关键事务MB5B / 自定义报表
一句话：MARD = 当前库存状态（现在），MARDH = 历史月末库存状态（过去某个时间点）

03 · MARDH 什么时候更新？（关键理解）
MARDH 不是每次收货、发货都写一条。它的更新逻辑是： 当 SAP 发生跨期间库存变化时，系统保存上一期间库存到 MARDH
STEP 1 — 6月30日
月初状态
MARD 物料A 库存 = 1000
MARDH 没有6月份记录
STEP 2 — 7月5日
跨月首笔操作
MB1A / MIGO 发料减少 100
库存变 = 900
STEP 3 — 系统检测
自动写入历史
系统发现 6→7 跨期间
自动把6月底库存写入 MARDH
年份=2026 期间=06 库存=1000
关键结论：MARDH 记录的是"期间切换时的历史快照"，不是交易流水。7月份即使每天发生100次收发货，MARDH 也不会生成100条记录。

05 · 为什么 MARDH 为空？5大原因全排查（按概率排序）
很多人以为"6月底 SAP 会自动把所有物料库存写入 MARDH"——这是错误的。 SAP 的真实逻辑：只有发生库存期间切换 + 后续库存业务，才可能触发 MARDH 更新。
1
MM 期间还没打开/切换（最常见）
现实已是7月中旬，但 SAP 当前期间仍停留在 2026/06。SAP 认为还在6月期间 → 不会生成 MARDH。
检查：MMSC 或 MMRV 查当前物料期间 → 解决：MMPV 执行期间关闭（2026/06 → 2026/07），再发生库存业务即可能生成 MARDH
2
物料在6月底以后没发生任何库存移动（也很常见）
6月30日库存 1000，7月份没有 101/601/261/311 等任何移动 → MARD 还是 1000，但 MARDH 为空（SAP 没触发历史保存）。
验证：MB51 查物料A在 2026.07.01-07.31 是否有物料凭证 → 无凭证则 MARDH 为空属正常
3
查错了库存层级（药企高频错误）
MARDH 是 物料 + 工厂 + 库存地点 粒度。只按 MATNR=A 查可能查不到 → 必须同时带 MATNR + WERKS + LGORT + LFGJA + LFMON。
4
该物料6月底库存为 0
6月采购 1000 → 生产领用 1000 → 月底库存 0。MARDH 通常不保存零库存记录（SAP 认为无需存历史）。
5
系统配置未维护历史库存 / 系统刚上线
检查：OMSY（物料管理开始期间）、OMWN（库存管理设置）、表 T001B（库存期间控制）。某些企业为性能会关闭历史更新；新上线系统历史数据未迁移。

05C · 业务强制要查历史库存？两种方案
方案一：MB5B（临时查询）
适合：临时查某月月底库存
限制：无批次维度；速度慢于查表
用法：输入公司代码/工厂/物料/截止日期
✓ 方案二：建立库存快照表（推荐）
每晚抽取：MARD + MCHB（批次）+ MSLB（特殊库存）+ MKPF/MSEG
保存格式：日期 + 物料 + 批次 + 数量
适合 BI 分析：月末库存 / 库龄 / 批次趋势 / 周转率
为什么推荐：MARDH 天然不足——无批次维度、不自带月末快照、不适合复杂分析
