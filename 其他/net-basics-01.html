2网络为什么存在？本质是「让设备之间可靠、安全地通信」
以 MES「投料确认」为例
MES 客户端（车间工位机）
↓ 发送请求
↓ 网络传输
↓ MES 服务器接收
↓ 访问数据库
↓ 返回结果
任何一个环节出问题 → 用户看到的就是
"MES 打不开"
所以「系统打不开」≠「开发 Bug」。先判断是哪一环，再决定找谁。
3企业网络四个核心区域（IT 经理必须懂）
① 办公网
财务、行政的电脑，访问 SAP GUI / OA / 邮件。
特点：开放性较高
② 生产网络 ⭐⭐⭐⭐⭐
MES 工位机、PLC、SCADA、DCS。贵司所在行业特别重要。
强调 IT / OT 隔离：办公室电脑 ✕ 生产设备，不能随意互通
③ 服务器区
SAP 应用 + 数据库、MES / LIMS / QMS、接口服务器。
通常放置于机房服务器区
④ DMZ 隔离区
对外服务的「隔离缓冲区」。例如供应商门户（SRM）：供应商 → 互联网 → 防火墙 → DMZ 服务器 → 内部系统。关键是不能让外部人员直接进入内部网络。
4网络通信最重要的三个概念 第一阶段核心
IP 地址
设备在网络中的「身份证」
SAP 服务器：192.168.10.20
MES 服务器：192.168.20.30
两系统通信 = 192.168.20.30 访问 192.168.10.20
SAP 场景：你知道每个应用服务器的 IP，是排错第一步。
端口
IP = 哪台电脑；端口 = 电脑上的哪个服务
MES 服务器 192.168.20.30 上：
· MES 服务 → 8080
· 数据库 → 1521
MES 访问库 = 192.168.20.30:1521
SAP 场景：GUI 连接不是「连 SAP」，而是连 SAP Dispatcher 端口 → 实例。
DNS
把名字翻译成 IP（人不喜欢记 IP）
访问 www.baidu.com → DNS 转成 IP → 服务器。企业内部常用 mes.company.com 代替 192.168.20.30。
SAP 场景：客户端配主机名，靠 DNS 解析；DNS 挂了，输 IP 能通、输名字不通。
5网络排查思维（IT 经理最重要） 五步逐层法
用户说「MES 打不开」，不要马上找开发
用户电脑 → 网络 → 服务器 → 应用 → 数据库 （自下而上逐层排查）
1
电脑是否联网？
先确认客户端本身在线
ping 网关
2
能否访问服务器？
网络层是否通到 MES 服务器
ping MES服务器IP
3
端口是否开放？
服务端口是否被防火墙挡住
telnet IP 端口
4
应用是否正常？
MES 服务进程是否起得来
检查 MES 服务
5
数据库是否正常？
最后确认底层数据层
检查 DB
